# PRD: 댓글거울 (Comment Mirror)

## 1. 목표
YouTube 영상 URL 하나만 넣으면 댓글을 자동 분석해서, 크리에이터에게 "잘하는 점 / 개선해야 할 점 / 다음 영상 액션"을 정리한 한 페이지 리포트를 만들어 준다. 로그인·서버 없음. 사용자 본인 API key 사용(BYOK).

## 2. 사용자 페르소나

### P1. 1인 크리에이터 "재호" (주 타깃)
- 30대 초반, 채널 구독자 1만 명, 영상당 댓글 200~500개
- 댓글을 다 읽을 시간이 없고, 어떤 부분이 시청자에게 잘 먹히는지 감으로만 안다
- 기술 친화적 (개발자/마케터 출신) — API key 발급 안내를 따라갈 수 있음
- "오늘 올린 영상이 어떤지 자기 전에 빠르게 보고 싶다"

### P2. 채널 운영 보조자 "수민" (보조 타깃)
- 대형 채널의 PD/AD. 본인 채널 아님
- 매주 영상 5~10편의 반응을 PM에게 보고해야 함
- 분석 결과를 캡처하거나 다운로드해 회의 자료로 씀

### P3. 단발성 호기심 사용자 "민지" (제외)
- 한 번 써보고 안 돌아옴. 페르소나로 잡지만 최적화 대상은 아님

## 4. 핵심 기능 (MVP)

### F1. URL 입력 → 리포트 자동 생성 [P0]
- 입력: `https://www.youtube.com/watch?v=...`, `https://youtu.be/...`, `https://www.youtube.com/shorts/...` 형식 허용
- 검증: 정규식 + videoId 11자 확인 + YouTube `videos.list`로 실재 여부 확인
- 거부: 채널 URL, 재생목록 URL, 잘못된 형식 → 즉시 인라인 에러

### F2. 댓글 수집 [P0]
- 최대 500개 (top-level만, 정렬: relevance)
- 100개 단위 페이지네이션, 진행률 표시
- 댓글이 0개 / 비활성화된 영상은 "분석할 댓글이 없습니다" 상태 표시

### F3. 감정 분석 (Sentiment) [P0]
- 긍정/중립/부정 3분류, 각 비율 0~100%
- 시각화: 가로 stacked bar + 숫자
- 결정성: temperature 0.2, 동일 입력 → 거의 동일 출력 (완벽한 결정성은 보장 안 함)

### F4. 토픽 추출 [P0]
- 댓글에서 자주 나오는 주제 묶음 5~10개
- 각 토픽에 sentiment(pos/neg/mixed), count, 대표 인용 2~3개 포함
- 인용은 원문 그대로(이모지 포함), 30자 이상은 말줄임

### F5. 크리에이터 피드백 [P0]
- **잘하는 점**: 3~5개, 구체적 행동 단위 (예: "오프닝 3초 안에 본론 시작")
- **개선점**: 3~5개, 비판이 아닌 관찰 (예: "BGM 볼륨이 자막보다 큼")
- **다음 영상 액션**: 3~5개, 다음 영상에 바로 시도할 수 있는 형식 (동사로 시작)
- **총평**: 2~3문장 요약

### F6. 리포트 히스토리 [P0]
- LocalStorage `cm:reports`에 저장
- 사이드바: 최신순, 영상 썸네일 + 제목 + 분석 일자
- 클릭 시 `/report/[id]`로 이동
- 개별 삭제 / 전체 삭제 가능

### F7. 설정 페이지 [P0]
- YouTube Data API key 입력 / 수정 / 삭제
- Anthropic API key 입력 / 수정 / 삭제
- key 발급 가이드 외부 링크 (각 key 옆 "발급 방법" 링크)
- 마스킹 표시 (`sk-ant-...****1234`)

### F8. 진행 상태 표시 [P0]
- 단계: `pending → validating_url → fetching_video → fetching_comments (N/500) → analyzing (M/M) → done | error`
- 진행률 바 + 텍스트 라벨
- 예상 소요 시간 표기 ("약 1~3분")

### F9. 분석 취소 [P1]
- 진행 중 "취소" 버튼 → AbortController로 in-flight fetch 중단
- 취소된 리포트는 저장하지 않음

## 5. MVP 제외 사항
- 로그인, 회원가입, 사용자 계정
- 서버, 데이터베이스, 백엔드 라우트
- 채널 전체 분석 (영상 단위만)
- 답글(reply) 분석
- 자동 번역 (댓글 언어 그대로 분석)
- 결제, 유료 플랜, 사용량 게이팅
- 리포트 공유 URL (서버가 필요해서 제외)
- 알림(이메일/푸시)
- 다중 영상 일괄 분석
- 분석 결과 비교 (영상 A vs B)
- 라이브 스트림 채팅 분석
- 모바일 앱

## 6. 지원 가능 영상 조건

| 조건 | 지원 | 비고 |
|------|------|------|
| 공개 영상 | ✅ | 기본 |
| 일부공개 (unlisted) | ✅ | URL이 있으면 댓글 API 접근 가능 |
| 비공개 (private) | ❌ | "비공개 영상은 분석할 수 없습니다" |
| 댓글 비활성화 | ❌ | "이 영상은 댓글이 비활성화되어 있습니다" |
| 댓글 0개 | ❌ | "분석할 댓글이 없습니다" |
| 연령 제한 | ⚠️ | API key 발급자 본인 인증 상태에 따라 다름 — 시도 후 에러 처리 |
| Shorts | ✅ | 일반 영상과 동일하게 처리 |

## 7. 비기능 요구사항

### 7.1 성능
- 분석 완료까지 95퍼센타일 3분 이내 (댓글 500개 기준)
- 첫 페이지 로드 (Lighthouse FCP) 1.5초 이내
- 리포트 페이지 렌더 (인터랙티브까지) 1초 이내

### 7.2 비용 (사용자 관점)
- 분석 1회당 예상 비용: $0.01~$0.05 (Claude Haiku 4.5 기준, 댓글 500개)
- 분석 시작 전 예상 비용을 화면에 표시 ("예상 비용: 약 $0.03")
- 실측 비용은 응답의 `usage` 필드로 계산해 리포트에 표시

### 7.3 정확도 / 품질
- 한국어, 영어 댓글에 대해 1차 품질 보장
- 그 외 언어는 best-effort (정확도 보장 안 함)
- 사르카즘 / 비꼬는 댓글은 오분류 가능성 명시 — 리포트에 "이 분석은 자동 생성이며 부정확할 수 있습니다" 디스클레이머 표시
- 인용은 반드시 원문 그대로 (LLM이 댓글을 새로 지어내는 것 금지 — 검증 단계 필요)

### 7.4 접근성
- WCAG AA 색상 대비
- 키보드만으로 모든 기능 사용 가능
- `prefers-reduced-motion` 존중 (fade-in 끔)
- 스크린리더 호환 (시맨틱 HTML, ARIA live region으로 진행 상태 알림)

### 7.5 보안 / 프라이버시
- API key는 LocalStorage에만 저장, 절대 콘솔/네트워크 페이로드/에러 메시지로 노출 금지
- 댓글 본문은 HTML 이스케이프 후 렌더 (XSS 방지)
- 외부 분석 도구(Google Analytics 등) 사용 안 함 — 트래픽 0
- 제3자 CDN으로 외부 폰트 로드 안 함 (시스템 폰트만)

### 7.6 브라우저 / 기기
- 데스크톱 우선 (Chrome / Safari / Firefox 최신 2버전)
- 모바일 웹은 reading-only 모드로 동작 (분석 가능하지만 입력은 데스크톱 권장)
- IE / Edge Legacy 지원 안 함

### 7.7 다국어 (i18n)
- 인터페이스 언어: 한국어 only (MVP)
- 댓글은 원문 언어 그대로 분석 (한/영 외 best-effort)
- 향후 영어 인터페이스 추가 가능성 열어둠

## 8. 사용자 경험 흐름

### 8.1 첫 방문 (Onboarding)
1. 메인 페이지 진입 → API key 없음 감지
2. "시작하려면 API key 2개가 필요합니다" 안내 카드 표시
3. "발급 가이드" 링크 → 외부 문서 (YouTube/Anthropic 공식)
4. 설정 페이지에서 key 입력 → 저장 시 형식 검증
   - YouTube key: `AIza`로 시작 + 39자
   - Anthropic key: `sk-ant-` 접두사
5. 검증 통과 시 메인 페이지로 이동

### 8.2 일반 흐름
1. 메인 URL 입력창에 영상 링크 붙여넣기
2. (자동) URL 검증 → videoId 추출 → `videos.list`로 영상 메타데이터 미리보기 (썸네일+제목)
3. "예상 비용 $0.03 · 댓글 약 N개" 안내
4. "분석 시작" 버튼 클릭
5. 진행 상태 표시 (단계별 progress)
6. 완료 시 자동으로 `/report/[id]`로 이동
7. 사이드바에 새 리포트 추가됨

### 8.3 에러 흐름
- URL 형식 오류 → 입력창 아래 인라인 빨간 메시지
- 영상 없음 → 모달 또는 토스트 "영상을 찾을 수 없습니다"
- 댓글 없음/비활성화 → "분석할 댓글이 없습니다 (이유: ...)"
- API key 만료/무효 → 설정 페이지로 유도 + 메시지
- 쿼터 초과 → "오늘 YouTube API 쿼터를 모두 사용했습니다. 내일 다시 시도하세요" + 카운트다운
- 네트워크 끊김 → "네트워크 연결을 확인해주세요" + 재시도 버튼
- Claude 응답 파싱 실패 → "분석 결과를 해석하지 못했습니다. 다시 시도하시겠습니까?" (1회 자동 재시도 후 표시)
- LocalStorage 한도 도달 → "저장 공간이 부족합니다. 이전 리포트를 삭제해주세요" + 가장 오래된 N개 자동 정리 옵션

### 8.4 빈 상태
- 리포트 0개일 때 메인 페이지: "첫 영상을 분석해보세요" + URL 입력창 강조
- 사이드바 비어있을 때: "분석한 리포트가 여기에 표시됩니다"

## 9. 콘텐츠 정책

### 9.1 혐오·욕설 댓글 처리
- 분석에는 포함 (필터링하면 부정 비율이 왜곡됨)
- 단, 리포트에 **인용으로 노출할 때**는 비속어를 `*` 마스킹
- 명백한 혐오 표현(차별, 폭력 위협)은 인용 대상에서 제외 (Claude에게 프롬프트로 지시)

### 9.2 스팸 댓글
- "구독해주세요", "맞구독", 이모지 도배, URL 광고 등은 분석 전 휴리스틱 필터링
- 필터링된 댓글 수는 리포트에 표시 ("스팸으로 추정되어 제외: N개")

### 9.3 작성자 프라이버시
- 인용 시 작성자 이름은 표시하지 않음 (또는 첫 글자만, `김**`)
- 프로필 사진 표시 안 함

## 10. 데이터 보존 및 삭제 정책

| 데이터 | 저장 위치 | 보존 기간 | 사용자 삭제 방법 |
|--------|----------|-----------|------------------|
| API key | LocalStorage | 사용자가 지울 때까지 | 설정 페이지 "삭제" 버튼 |
| 리포트 | LocalStorage | 사용자가 지울 때까지 | 사이드바 개별 삭제 / 전체 초기화 |
| 댓글 원문 | LocalStorage (리포트 내 인용분만) | 리포트와 동일 | 리포트 삭제 시 함께 |

- LocalStorage가 5MB 한도에 도달하면: 사용자에게 알림 + 가장 오래된 리포트 자동 후보 제시 (사용자 확인 후 삭제)
- 브라우저 캐시를 사용자가 지우면 모든 데이터가 사라짐 — 첫 방문 시 면책 고지

## 11. 사용자 면책 / 책임 고지
- API 호출 비용은 사용자 본인 부담
- 분석 결과는 자동 생성이며 100% 정확하지 않음
- 본 서비스는 YouTube 또는 Anthropic의 공식 제품이 아님
- 분석 결과를 외부 공개 시 작성자 프라이버시 주의

## 12. 성공 지표 (Phase 1, 비공식)
- **북극성 지표**: 주간 활성 사용자 1인당 평균 분석 리포트 수
- 보조 지표:
  - 분석 1회당 사용자 인지 시간 (시작 → done)
  - 분석 완료율 (시작 대비 done 비율)
  - 사이드바에서 과거 리포트 재방문율
- 측정 방법: 로컬 콘솔 로그 디버그 모드 + 사용자 인터뷰 (자동 텔레메트리 없음)
